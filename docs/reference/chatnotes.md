# Chat Notes

## Purpose
This file is the long-term collaboration memory for the `Slowdrag website building` workspace.

It exists to:
- log concise summaries of the conversations held in this workspace
- preserve recurring user preferences, feedback patterns, and decision logic
- help future sessions infer the user's working style and thought process without relying on hidden reasoning
- complement `archetype.md`, which remains the canonical project-memory file

This file should capture interaction patterns, decision summaries, and clear preferences.
It should not store private chain-of-thought.

## How To Use This File
- Read this file at the start of every new session together with `archetype.md`.
- Update it when the user's preferences, tone expectations, review style, or decision-making patterns become clearer.
- Add concise session notes after meaningful exchanges.
- Prefer durable observations over raw transcript dumping.
- When older assumptions are disproved, update the current sections directly and note the shift in `Preference Changes`.

## User Collaboration Profile
- The user works like a creative director and expects the assistant to operate as an active strategic and execution partner.
- The user prefers step-by-step progression, but not passive planning; they want visible execution between reviews.
- The user wants the system to infer taste and spacing issues proactively rather than waiting for explicit correction.
- The user expects strong point of view, high intention, and polished execution rather than safe default web design.
- The user values memory systems and wants the workspace to accumulate understanding across sessions.

## Communication Preferences
- Be direct, collaborative, and constructive.
- Avoid generic reassurance or vague design language.
- Summaries should be concise and high-signal.
- When a direction is weak, acknowledge it cleanly and pivot instead of defending it.
- Do not make the user repeat settled context if it can be stored and reused.
- The user wants a clear wake-up ritual so they can re-enter the workspace and restore shared context quickly.

## Thought Process Patterns
- The user makes decisions through iterative visual critique rather than long upfront specification.
- The user often reacts strongly to whether something feels intentional, sleek, and agency-grade.
- The user tends to refine by contrast:
  - less clustered
  - less generic
  - less minimal when it feels empty
  - less decorative when it lacks purpose
- The user prefers systems that can evolve with the project, including reusable memory files and auto-updating content behavior where possible.
- The user is highly sensitive to rhythm, spacing, motion quality, and whether an interaction translates the intended idea accurately.

## Aesthetic And Product Preferences
- The work should feel like a serious creative studio, not a template portfolio.
- Reference-led thinking is welcome, but visible copying is not.
- Motion and interaction should feel authored, not ornamental.
- Typography spacing matters a lot, especially in headers and display text.
- The user likes unusual but legible interactive ideas when they clearly support the concept.
- Sleekness is preferred over clutter when showcasing a high volume of work.
- Dense asset displays should still feel edited and controlled.
- In asset-heavy design showcases, the user prefers raw images as they are rather than framed tiles with artificial backgrounds.
- Sparse logo-only or visually thin assets should be filtered out when they weaken the overall field.
- In mixed design streams, branding-heavy material should lead over weaker supporting assets.
- Tool-like visual metaphors, such as a Photoshop-style selected area, are welcome when they sharpen the concept instead of becoming decoration.
- In looping image streams, the user prefers fewer, larger assets per line rather than dense rows of small images.
- Repetition across adjacent lines weakens the section; each lane should feel distinct.
- Repetition caused by visible loop cloning is also undesirable; continuous motion should feel seamless without obvious duplicate copies entering view.
- When the user shares a reference website for motion, they want the interaction grammar translated into Slow Drag's context, not a surface-level visual imitation.
- For design showcases, a curated scroll-led sequence can be preferable to an infinite loop if it feels more authored and less repetitive.
- When a design interaction starts feeling overworked, the user is comfortable rolling back several steps to a cleaner earlier state instead of iterating forever on the wrong layer.
- The user responds well when typography itself participates in the interaction language, as long as the effect is precise and not decorative noise.
- The user is open to shifting from interactive typography to atmospheric typography if the latter better suits the section mood.

## Feedback Signals To Watch
- Strong negative reactions usually indicate a mismatch in intention, not just style.
- Phrases like `horrible`, `scrap it`, `wipe out everything`, or `did not get the full intention translated` mean the design logic has missed the concept and needs a reset rather than minor polishing.
- Requests such as `analyse yourself`, `understand it yourself`, or `do not make such mistakes further` signal that the assistant should infer more and rely less on obvious defaults.
- When the user says something is `clustered`, `generic`, or `too minimal`, reduce noise while increasing intentional composition.

## Current Collaboration Heuristics
- Preserve what the user explicitly likes and build around it.
- When introducing new layout systems, make sure the concept is legible at first glance.
- Treat every section as a brand-positioning surface, not just a content container.
- Keep memory files updated so future sessions can continue from the user's established taste and decision patterns.
- When the user enters `WAKE UP`, reload the memory files first and respond with a concise orientation summary before moving into new work.
